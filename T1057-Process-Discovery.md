# T1057 – Process Discovery

## Overview

Process Discovery (T1057) is a common reconnaissance technique used by attackers to enumerate running processes on a compromised system. By identifying active processes, adversaries can locate security software, privileged applications, or processes suitable for credential theft, code injection, or privilege escalation.

This detection demonstrates how process discovery activity can be identified using Sysmon process creation telemetry and validated within Splunk Enterprise after simulating attacker behaviour with Atomic Red Team.

---

## MITRE ATT&CK

| Field | Value |
|--------|-------|
| Technique ID | T1057 |
| Technique | Process Discovery |
| Tactic | Discovery |

---

## Detection Objective

The objective of this detection is to identify process discovery activity on a Windows endpoint using Sysmon Process Creation (Event ID 1). Commands such as `tasklist.exe` are frequently used by attackers to enumerate running processes after gaining access to a system.

---

## Attack Simulation

The Atomic Red Team framework was used to simulate process discovery using the following Atomic Test:

```powershell
Invoke-AtomicTest T1057 -TestNumbers 2
```

### Expected Result

- `tasklist.exe` is executed.
- Sysmon generates Process Creation (Event ID 1).
- Events are forwarded to Splunk Enterprise.
- The detection query identifies the simulated activity.

---

## Telemetry Generated

| Source | Event |
|---------|-------|
| Sysmon | Event ID 1 – Process Creation |

The primary telemetry source was Sysmon Process Creation (Event ID 1), which recorded execution of `tasklist.exe` together with the command line, parent process, user account, and execution timestamp.

---

## Detection Query

```spl
index=* "tasklist.exe"
| rex field=_raw "<Data Name='Image'>(?<Image>[^<]+)</Data>"
| rex field=_raw "<Data Name='CommandLine'>(?<CommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentImage'>(?<ParentImage>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentCommandLine'>(?<ParentCommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='User'>(?<User>[^<]+)</Data>"
| table _time User Image CommandLine ParentImage ParentCommandLine
```

---

## Detection Logic

The detection searches Sysmon process creation events for executions of `tasklist.exe`. Since the Sysmon Technology Add-on was unavailable, XML fields were manually extracted from raw Sysmon events using the `rex` command.

The query extracts:

- User
- Process Image
- Command Line
- Parent Process
- Parent Command Line

These fields provide context to validate process discovery activity and identify the originating process responsible for executing the command.

---

## Detection Validation

The detection successfully identified execution of `tasklist.exe` generated during the Atomic Red Team simulation.

Validation confirmed:

- Execution of `tasklist.exe`
- Command line arguments
- Parent process
- Executing user
- Event timestamp
