# T1016 – System Network Configuration Discovery

## Overview

System Network Configuration Discovery (T1016) is used by attackers to gather information about a compromised system's network configuration, including IP addresses, DNS settings, network adapters, and firewall configurations. This information helps adversaries understand the network environment, identify potential lateral movement opportunities, and plan further attack activity.

This detection demonstrates how network configuration discovery can be identified using Sysmon process creation telemetry and validated within Splunk Enterprise after simulating attacker behaviour with Atomic Red Team.

---

## MITRE ATT&CK

| Field | Value |
|--------|-------|
| Technique ID | T1016 |
| Technique | System Network Configuration Discovery |
| Tactic | Discovery |

---

## Detection Objective

The objective of this detection is to identify network configuration discovery activity on a Windows endpoint using Sysmon Process Creation (Event ID 1). Commands such as `ipconfig.exe` and `netsh.exe` are commonly used by attackers to enumerate network settings during post-compromise reconnaissance.

---

## Attack Simulation

The Atomic Red Team framework was used to simulate network configuration discovery using the following Atomic Test:

```powershell
Invoke-AtomicTest T1016 -TestNumbers 1
```

### Expected Result

- `ipconfig.exe` and `netsh.exe` are executed.
- Sysmon generates Process Creation (Event ID 1).
- Events are forwarded to Splunk Enterprise.
- The detection query identifies the simulated activity.

---

## Telemetry Generated

| Source | Event |
|---------|-------|
| Sysmon | Event ID 1 – Process Creation |

The primary telemetry source was Sysmon Process Creation (Event ID 1), which recorded execution of `ipconfig.exe` and `netsh.exe` along with the command line, parent process, user account, and execution timestamp.

---

## Detection Query

```spl
index=* ("ipconfig.exe" OR "netsh.exe")
| rex field=_raw "<Data Name='Image'>(?<Image>[^<]+)</Data>"
| rex field=_raw "<Data Name='CommandLine'>(?<CommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentImage'>(?<ParentImage>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentCommandLine'>(?<ParentCommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='User'>(?<User>[^<]+)</Data>"
| table _time User Image CommandLine ParentImage ParentCommandLine
| sort -_time
```

---

## Detection Logic

The detection searches Sysmon process creation events for executions of `ipconfig.exe` or `netsh.exe`. Since the Sysmon Technology Add-on was unavailable, XML fields were manually extracted from raw Sysmon events using the `rex` command.

The query extracts:

- User
- Process Image
- Command Line
- Parent Process
- Parent Command Line

These fields provide context to validate network configuration discovery activity and support further investigation.

---

## Detection Validation

The detection successfully identified execution of `ipconfig.exe` and `netsh.exe` generated during the Atomic Red Team simulation.

Validation confirmed:

- Execution of `ipconfig.exe`
- Execution of `netsh.exe`
- Command line arguments
- Parent process
- Executing user
- Event timestamp
