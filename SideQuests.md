# Side Quests

While building this detection engineering project, I explored several offensive security tools and techniques to better understand attacker behaviour and the telemetry generated by common security activities.

These experiments were completed in a controlled lab environment and were not part of the core detection engineering pipeline. Instead, they helped reinforce concepts related to reconnaissance, exploitation, and post-exploitation that defenders commonly monitor.

---

# Nmap Reconnaissance

## Objective

Develop familiarity with network reconnaissance techniques and understand how exposed services can be identified during the early stages of an attack.

## TCP Port Discovery

Performed a scan of the most common TCP ports to identify publicly accessible services.

**Command**

```bash
nmap --top-ports 10 45.33.32.156
```

### Results

<img width="581" height="571" alt="nmap recon" src="https://github.com/user-attachments/assets/de41135a-3020-4fec-988b-125fb4ad131e" />


### Observations

- Identified SSH (22/tcp) and HTTP (80/tcp) as open services.
- Multiple commonly targeted ports were filtered by the host or firewall.
- Demonstrated how attackers rapidly enumerate exposed services before further investigation.

---

## Service Version Enumeration

Performed service detection against the discovered ports to identify running software and operating system information.

**Command**

```bash
nmap -sV -p 22,80 45.33.32.156
```

### Results

<img width="807" height="272" alt="nmap open ports services" src="https://github.com/user-attachments/assets/232f79c1-7c76-4d69-adbb-ceadf5375149" />


### Observations

- Detected OpenSSH 6.6.1p1.
- Detected Apache HTTP Server 2.4.7.
- Identified the target as an Ubuntu Linux system.
- Demonstrated how service fingerprinting provides valuable information for both attackers and defenders.

# Netcat

## Objective

Gain practical experience using Netcat as a lightweight networking utility for connectivity testing and service interaction.

## Connectivity Verification

Verified that the SSH service was accepting TCP connections.

**Command**

```bash
nc -vz 45.33.32.156 22
```

### Observations

- Successfully established a TCP connection.
- Confirmed that the SSH service was reachable before conducting further enumeration.

---

# Key Takeaways

These side explorations complemented the primary detection engineering project by improving my understanding of:

- Network reconnaissance techniques.
- Service discovery and enumeration.
- TCP connectivity testing.
- How attackers gather information before exploitation.
- How defensive teams can monitor reconnaissance activity as an early indicator of compromise.

Although these activities were not directly used to develop the Splunk detections in this repository, they provided valuable context for understanding attacker methodology and defensive monitoring.
