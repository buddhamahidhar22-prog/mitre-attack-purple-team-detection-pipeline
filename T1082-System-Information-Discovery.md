# T1082 – System Information Discovery

## Overview

System Information Discovery (T1082) is used by attackers to gather information about a compromised system, including the operating system, version, hardware configuration, and installed software. This information helps adversaries understand the environment, identify potential privilege escalation opportunities, and tailor subsequent attack techniques.

This detection demonstrates how system information discovery can be identified using Sysmon process creation telemetry and validated within Splunk Enterprise after simulating attacker behaviour with Atomic Red Team.

---

## MITRE ATT&CK

| Field | Value |
|--------|-------|
| Technique ID | T1082 |
| Technique | System Information Discovery |
| Tactic | Discovery |

---

## Detection Objective

The objective of this detection is to identify system information discovery activity on a Windows endpoint using Sysmon Process Creation (Event ID 1). Detecting commands such as `systeminfo.exe` can provide visibility into reconnaissance activities commonly performed by attackers after gaining initial access.

---

## Attack Simulation

The Atomic Red Team framework was used to simulate system information discovery using the following Atomic Test:

```powershell
Invoke-AtomicTest T1082 -TestNumbers 1
```

### Expected Result

- `systeminfo.exe` is executed.
- Sysmon generates Process Creation (Event ID 1).
- Events are forwarded to Splunk Enterprise.
- The detection query identifies the simulated activity.

---

## Telemetry Generated

| Source | Event |
|---------|-------|
| Sysmon | Event ID 1 – Process Creation |

During testing, an additional Sysmon Event ID 13 (Registry Value Set) was observed. This event was generated by normal Windows activity following execution of the Atomic Test and was not used as the primary detection source. The detection focused on Sysmon Event ID 1, which directly recorded execution of `systeminfo.exe`.

---

## Detection Query

```spl
index=* "systeminfo.exe"
| rex field=_raw "<Data Name='Image'>(?<Image>[^<]+)</Data>"
| rex field=_raw "<Data Name='CommandLine'>(?<CommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentImage'>(?<ParentImage>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentCommandLine'>(?<ParentCommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='User'>(?<User>[^<]+)</Data>"
| table _time User Image CommandLine ParentImage ParentCommandLine
| sort -_time
```

---

## Detection Logic

The detection searches Sysmon process creation events for executions of `systeminfo.exe`. Since the Sysmon Technology Add-on was unavailable, XML fields were manually extracted from raw Sysmon events using the `rex` command.

The query extracts:

- User
- Process Image
- Command Line
- Parent Process
- Parent Command Line

These fields provide sufficient context to validate system information discovery activity and support further investigation.

---

## Detection Validation

The detection successfully identified execution of `systeminfo.exe` generated during the Atomic Red Team simulation.

Validation confirmed:

- Execution of `systeminfo.exe`
- Command line arguments
- Parent process
- Executing user
- Event timestamp
