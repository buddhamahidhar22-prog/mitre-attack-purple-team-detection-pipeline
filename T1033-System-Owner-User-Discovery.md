# T1033 – System Owner/User Discovery

## Overview

System Owner/User Discovery (T1033) is used by attackers to identify the currently logged-in user or active user sessions on a compromised system. This information helps adversaries understand user privileges, identify potential targets for credential theft, and determine the context of the compromised endpoint.

This detection demonstrates how user discovery activity can be identified using Sysmon process creation telemetry and validated within Splunk Enterprise after simulating attacker behaviour with Atomic Red Team.

---

## MITRE ATT&CK

| Field | Value |
|--------|-------|
| Technique ID | T1033 |
| Technique | System Owner/User Discovery |
| Tactic | Discovery |

---

## Detection Objective

The objective of this detection is to identify user discovery commands executed on a Windows endpoint using Sysmon Process Creation (Event ID 1). Commands such as `whoami.exe` are commonly used by attackers during post-compromise reconnaissance.

---

## Attack Simulation

The Atomic Red Team framework was used to simulate user discovery using the following Atomic Test:

```powershell
Invoke-AtomicTest T1033 -TestNumbers 1
```

### Expected Result

The Atomic test executes several user discovery commands including:

- `whoami`
- `query user`
- `query session`

During testing, WMIC-based commands failed because WMIC is deprecated on modern Windows 11 systems. The remaining commands executed successfully and generated the expected telemetry.

---

## Telemetry Generated

| Source | Event |
|---------|-------|
| Sysmon | Event ID 1 – Process Creation |

The primary telemetry source was Sysmon Process Creation (Event ID 1), which recorded execution of `whoami.exe` along with the associated command line, parent process, user, and timestamp.

---

## Detection Query

```spl
index=* "whoami.exe"
| rex field=_raw "<Data Name='Image'>(?<Image>[^<]+)</Data>"
| rex field=_raw "<Data Name='CommandLine'>(?<CommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentImage'>(?<ParentImage>[^<]+)</Data>"
| rex field=_raw "<Data Name='ParentCommandLine'>(?<ParentCommandLine>[^<]+)</Data>"
| rex field=_raw "<Data Name='User'>(?<User>[^<]+)</Data>"
| table _time User Image CommandLine ParentImage ParentCommandLine
| sort -_time
```

---

## Detection Logic

The detection searches Sysmon process creation events for executions of `whoami.exe`. Since the Sysmon Technology Add-on was unavailable, XML fields were manually extracted from raw Sysmon events using the `rex` command.

The query extracts:

- User
- Process Image
- Command Line
- Parent Process
- Parent Command Line

These fields provide context to validate user discovery activity and identify the process responsible for executing the command.

---

## Detection Validation

The detection successfully identified execution of `whoami.exe` generated during the Atomic Red Team simulation.

Validation confirmed:

- Execution of `whoami.exe`
- Command line arguments
- Parent process (`cmd.exe` or `powershell.exe`)
- Executing user
- Event timestamp
